3️⃣ THEME-AWARE RENDERING (MANDATORY)
DO NOT:

Apply CSS filter: invert

Change SVG fill colors

Apply opacity hacks

Let Tailwind auto-theme it

MUST:

Explicitly switch logo file by theme

Example (React / Next.js):

const theme = useTheme(); // dark | light

<img
  src={`/logos/aws/aws-${theme}.svg`}
  alt="AWS"
  className="h-6 w-auto"
/>


This logic must be reused everywhere logos appear:

Hero cards

Platform sections

Partner page

Solutions

Footer

AIX / DevX / BillOps pages

4️⃣ VISIBILITY GUARANTEE (LOGO CONTAINERS)

Every logo must be wrapped in a visibility-safe container:

.logo-pill {
  background: rgba(255,255,255,0.95); /* for dark mode */
  border-radius: 10px;
  padding: 10px 14px;
}


Light mode equivalent:

[data-theme="light"] .logo-pill {
  background: rgba(0,0,0,0.05);
}


This is non-negotiable.

5️⃣ GLOBAL LOGO REGISTRY (SINGLE SOURCE OF TRUTH)

Create:

/client/src/config/logos.ts


Example:

export const LOGOS = {
  aws: {
    light: "/logos/aws/aws-light.svg",
    dark: "/logos/aws/aws-dark.svg",
  },
  azure: {
    light: "/logos/azure/azure-light.svg",
    dark: "/logos/azure/azure-dark.svg",
  },
  openai: {
    light: "/logos/openai/openai-light.svg",
    dark: "/logos/openai/openai-dark.svg",
  },
};


All pages must import from here.
No inline paths. No duplication.
